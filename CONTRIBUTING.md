# 贡献指南（CONTRIBUTING.md）

感谢你对 PictureView 的关注与贡献！为让贡献流程清晰高效，请参考下面的步骤。

1. 报告问题
- 若发现 bug，请先在仓库 Issues 中搜索是否已有相同问题。
- 提交新 issue 时请尽量包含：环境信息（操作系统、Qt 版本）、重现步骤、相关日志、截图（若适用）。

2. 提交代码（Pull Request）
- Fork 仓库并在本地新建分支（分支名示例：`fix/issue-123` 或 `feat/xxx`）。
- 保持提交粒度清晰，单个 PR 尽量只解决一件事。
- 在 PR 描述中包含：变更说明、关联问题号（例如 `Fixes #123`）、测试方法/复现步骤。
- 我们偏好使用 clang-format / 一致的代码风格（请跟随仓内已有格式）。

3. 代码审查与合并
- 所有 PR 会由维护者审查；可能会要求修改、补充单元测试或文档。
- 若 PR 修复安全问题，请在合并前通过私下渠道通知维护者以避免提前公开漏洞细节。

4. 本地构建与测试
- 请参阅 README.md 中的构建说明（Linux/Windows/macOS）。提交前请确保变更不会破坏基本构建流程。

非常感谢你的贡献 — 若有任何问题欢迎在 Issues 中讨论。
